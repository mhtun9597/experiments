    SumSub Verification Workflow

    Customer Service
    August 28, 2024 08:49 Updated

    We currently offer three types of verification: Level 1, Level 2, and EDD Check (Please check the relevant article: Necessary Documents For KYC Process)

    1. Level 1 Verification: For Level 1, clients are required to upload only their ID document.

    2. Level 2 Verification: In Level 2, if the address provided by the client does not match the address on the ID, the client must provide both their ID and Proof of Address (POA).

    3. EDD Check(only for official website clients): The Enhanced Due Diligence (EDD) Check requires clients to provide their ID, Proof of Address (POA), and complete a facial recognition process.

     

    Level 1 Verification

    1. User Registration

      • Step 1: User submits their registration information through the registration form.
      • Step 2: The system captures basic user details, such as name, email, phone number, and address.

    2. Initiate Verification

      • Step 1: The system determines that Level 1 verification is required.
      • Step 2: The system triggers a verification request to Sumsub's API for Level 1, requesting the user to upload their ID document.

    3. Document Submission

      • Step 1: User receives a request to upload their ID document through the verification portal.
      • Step 2: User uploads the required ID document.

    4. Document Verification

      • Step 1: Sumsub verifies the authenticity of the submitted ID document.
      • Step 2: Sumsub provides the verification result to the system.

    5. Validation Result

      • Step 1: The system processes the verification result.
      • Step 2: If verified, the user gains access to the platform’s features. If rejected, the system informs the user of the failure and provides instructions on how to resolve the issue.

    6. User Notification

      • Step 1: The system informs the user of the verification outcome via in-app notification.

    7. Compliance and Record Keeping

      • Step 1: Verification results and documents are securely stored in compliance with data protection regulations.

    Level 2 Verification

    1. User Registration

      • Step 1: User submits their registration information through the registration form.
      • Step 2: The system captures basic user details, such as name, email, phone number, and address.

    2. Determine Verification Level

      • Step 1: The system identifies that Level 2 verification is required because the (user’s address on the ID/ user declare that the address) does not match the provided address 

    3. Initiate Verification

      • Step 1: The system triggers a verification request to Sumsub's API for Level 2, requesting the user to upload their ID and Proof of Address (POA).

    4. Document Submission

      • Step 1: User receives a request to upload both their ID and Proof of Address (POA) through the verification portal.
      • Step 2: User uploads the required documents.

    5. Document Verification

      • Step 1: Sumsub verifies the authenticity of the ID and POA documents and checks for consistency between them.
      • Step 2: Sumsub provides the verification result to the system.

    6. Validation Result

      • Step 1: The system processes the verification result.
      • Step 2: If verified, the user gains access to the platform’s features. If rejected, the system informs the user of the failure and provides instructions on how to resolve the issue.

    7. User Notification

      • Step 1: The system informs the user of the verification outcome via in-app notification.

    8. Compliance and Record Keeping

      • Step 1: Verification results and documents are securely stored in compliance with data protection regulations.

     

    EDD Check

    1. User Registration

      • Step 1: User submits their registration information through the registration form.
      • Step 2: The system captures basic user details, such as name, email, phone number, and address.

    2. Determine Verification Level

      • Step 1: The system identifies that Enhanced Due Diligence (EDD) is required based on risk factors or regulatory requirements.

    3. Initiate Verification

      • Step 1: The system triggers a verification request to Sumsub's API for EDD, requesting the user to upload their ID, Proof of Address (POA), and complete facial recognition.

    4. Document and Facial Recognition Submission

      • Step 1: User receives a request to upload their ID, Proof of Address (POA), and to complete facial recognition through the verification portal.
      • Step 2: User uploads the required documents and takes a selfie for facial recognition.

    5. Document and Facial Recognition Verification

      • Step 1: Sumsub verifies the authenticity of the ID and POA documents.
      • Step 2: Sumsub performs facial recognition to match the selfie with the ID photo.
      • Step 3: Sumsub provides the verification result to the system.

    6. Validation Result

      • Step 1: The system processes the verification result.
      • Step 2: If verified, the user gains access to the platform’s features. If rejected, the system informs the user of the failure and provides instructions on how to resolve the issue.

    7. User Notification

      • Step 1: The system informs the user of the verification outcome via in-app notification.

    8. Compliance and Record Keeping

      • Step 1: Verification results and documents are securely stored in compliance with data protection regulations.
