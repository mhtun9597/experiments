# MH-VaultCX Knowledge Base

## Company Overview

### MH-VaultCX Overview
MH-VaultCX is a SaaS agentic platform that enables organizations to build, configure, and operate their own AI-powered customer interaction systems.

### MH-VaultCX Core Purpose
MH-VaultCX helps organizations create tailored AI agentic systems instead of relying on generic assistants.

### MH-VaultCX Primary Value
MH-VaultCX allows organizations to combine AI automation with human support in a single operational platform.

---

## Core Platform Capabilities

### Capability: Domain Management
MH-VaultCX allows organizations to create and manage their own AI domains.

### Domain Definition
In MH-VaultCX, a domain represents a specific assistant context, business function, or service area.

### Domain Configuration Scope
Each domain can have its own:
- prompt design
- knowledge sources
- tools
- model configuration
- agent behavior rules

### Domain Use Cases
Typical domain use cases include:
- customer support
- onboarding assistance
- internal knowledge assistant
- order inquiry assistant
- billing help assistant
- technical troubleshooting assistant

---

## Prompt Management

### Capability: Prompt Management
MH-VaultCX allows organizations to manage prompts used by their AI agents.

### Prompt Management Scope
Prompt management may include:

Prompt management allows each organization to shape how its AI behaves and responds.

## AI and Human Support Model

### Capability: AI Agent with Human Backup
Once an organization completes setup for domains, prompts, knowledge, tools, and model integrations, it can operate an AI-powered agentic system with human agent backup.

### AI and Human Support Flow
This operating model means:
- customers can first interact with AI
- AI can answer using configured knowledge and tools
- conversations can be handed off to a human agent when needed
- support continuity is preserved across AI and human assistance

### Operational Benefit
This model combines automation efficiency with service reliability.

## Customer Interaction Channels

### Channel: AI Chat Panel
Each registered organization receives access to an AI chat panel.

### AI Chat Panel Purpose
The AI chat panel allows customers to chat with the organization's AI agent.

### AI Chat Panel Behavior
The AI agent responds based on the configured domain, prompt, ingested knowledge, and integrated tools.

### AI Chat Panel Use Cases
Typical AI chat panel use cases include:
- asking product questions
- checking service information
- requesting general guidance
- obtaining instant support for common issues
- starting a workflow before escalation to human support

### Channel: Human Agent Panel
Each registered organization also receives a human agent panel.

### Human Agent Panel Purpose
The human agent panel allows human support agents to receive escalated conversations and continue support directly with customers.

### Human Agent Panel Functions
Human support agents can:
- receive escalated conversations
- speak directly with customers
- review conversation context
- continue unresolved issues
- provide manual assistance when AI is insufficient

### Human Agent Panel Role
The human agent panel acts as the operational fallback and live support layer of MH-VaultCX.

### Channel: Embeddable Chat Widget
MH-VaultCX provides an embeddable chat widget that organizations can place on their websites.

### Embeddable Chat Widget Purpose
The widget allows website visitors to start AI-powered conversations directly from the organization's website.

### Embeddable Chat Widget Benefits
The chat widget allows visitors to:
- ask questions directly from the website
- receive support without leaving the page
- be routed to human support when needed

### Embeddable Chat Widget Placement Examples
Common placement examples include:
- homepage
- support center
- product pages
- checkout pages
- dashboard portals
- contact pages

## Integration Process

### Integration Step: Initial Inquiry
The organization contacts MH-VaultCX for registration or integration interest.

### Integration Step: Requirement Discovery
The MH-VaultCX team reviews the organization's goals, service model, customer support workflows, and integration needs.

### Integration Step: Workspace Provisioning
A workspace is created for the organization after qualification or plan selection.

### Integration Step: Domain Setup
The organization creates one or more domains based on its service areas.

### Integration Step: Prompt and Knowledge Configuration
The organization configures prompts and uploads or connects its knowledge sources.

### Integration Step: Tool Integration
The organization implements or connects required tools such as CRM lookup, ticket creation, or email functions.

### Integration Step: Model Configuration
The organization selects or connects the preferred foundation model provider.

### Integration Step: Human Agent Setup
Human support users are onboarded to the human agent panel.

### Integration Step: Widget Deployment
The embeddable chat widget is installed on the organization's website or portal.

### Integration Step: Testing and Go-Live
The organization performs testing, validates AI and handoff behavior, and launches the experience to end users.

## Example Use Case

### Use Case: Website Customer Support
A customer visits an organization's website and opens the MH-VaultCX chat widget.

### Website Customer Support Flow
1. The customer asks a support question.
2. The AI agent responds using configured prompt instructions and retrieved knowledge.
3. If the issue requires a business action, the agent may use an integrated tool.
4. If the issue cannot be resolved by AI, the conversation can be handed off to a human agent.
5. The human agent continues the conversation through the human agent panel.

### Use Case Outcome
This creates a blended support experience powered by AI and backed by human assistance.
