    Terminate Client's Profile

    Customer Assistant
    March 03, 2026 07:37 Updated

    Based on the Verification stage at which the client's request is coming: 

    • Client Requests Account Deletion During Sumsub Verification / Wants to Change Upline
    • Existing Client Requests Account Deletion / No Longer Wants to Trade
    • Termination Request from Account Managers / Sales

    Terminate Pending Account

    Client Requests Account Deletion During Sumsub Verification / Wants to Change Upline

    1. Upon receiving a request to delete an account during the verification process, CS should advise the client to complete the account opening process first. Inform client that they can request a referrer change after registration.
    2. CS to clearly inform the client that one ID can only register one account and once the account is terminated, they cannot register a new account.
    3. If Client still wants deletion after warnings. CS should escalate the ticket to OB
    4. OB to perform Final Reject in Sumsub. CRM status automatically updates to "Client Termination".

    Terminate Approved Account

    Existing Client Requests Account Deletion / No Longer Wants to Trade

    1. CS to inform the client that we cannot delete CRM profiles, only disable them.
    2. Request must come from the client’s registered email.
      If ❌ → request client to email from their registered address
    3. If the request comes from client's registered account, check the following: 
      • No open positions
      • No open pending orders
      • No pending withdrawals
    4. If all conditions are met - Change status to "Client Termination" in CRM.
    5. OB records the deletion in the Changing File

    Termination Request from Account Managers / Sales

    1. CS to Reject requests not sent by the client and ask Sales to have the client email us from their registered address. If the request isn’t from the client, reject it and tell Sales the client must send it from their registered email. Only accept requests from the client. Otherwise, reject and ask Sales to have the client email us directly.
    2.  If Sales Insists, they should provide: 
      • BD approval
      • Supporting documents (reason, evidence, internal notes, etc.)
    3. If the above confirmations received, proceed with escalation the case to RC and Compliance. They will review the client’s history and decide if termination is necessary.
    4. If termination is approved: a notice email must be prepared and sent.
    5. OB to Change CRM status to "Client Termination".
    6. OB records the deletion in the Changing File
